## Trust Boundaries

| Boundary | Description |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-2-SC | Tampering | npm installs (`@angular/cli`, `sirv-cli`, Angular's own transitive scaffold deps) | medium | mitigate | 02-RESEARCH.md's Package Legitimacy Audit already verified `sirv-cli` and `@angular/cli` as "Approved" (no postinstall scripts, established repos, high download counts) — no `[ASSUMED]`/`[SUS]` verdict, no blocking checkpoint required. |
| T-2-02 | Information Disclosure | `install: npm ci --ignore-scripts` (this plan's literal choice) | high | mitigate | Declaring the install command with `--ignore-scripts` baked in (rather than relying on a runtime override) is what lets D2-04's env-stripped spawn (Plan 02-03) actually prevent a third-party postinstall hook from running with even the minimal allowlisted env. |
| T-2-04 | Tampering | `template` field value | low | mitigate | `template: stacks/angular/template` is a fixed, committed, author-controlled path — not derived from any run-time input — so `copyWorkspace` (Plan 02-03) never resolves a spec-supplied arbitrary path outside this known directory. |
</threat_model>
