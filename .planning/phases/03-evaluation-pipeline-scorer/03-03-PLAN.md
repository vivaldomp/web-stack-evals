<objective>
Implement the two "PNG-bytes-only" evaluators — the two of Phase 3's four evaluators that never touch a live browser page, only Buffer inputs and (for the judge) one network call. Both are pure `EvaluatorPort` implementations: `createPixelMatchEvaluator()` (EVAL-01) and `createJudgeEvaluator()` (EVAL-04).

Purpose: PixelMatch is the deterministic, zero-network sub-score every fixture-driven test in this phase can exercise instantly. The judge is the one evaluator with a network dependency and the one file besides tests permitted to import `@earendil-works/pi-ai` — its test-double path (`fauxProvider()`) is what keeps the "no agent, no network" checkpoint (ROADMAP SC5) actually reachable in CI.
Output: `src/eval/pixelmatchEvaluator.ts`, `src/eval/judgeEvaluator.ts`, and `tests/fixtures/eval/pngFixtures.ts` (procedurally-generated PNG buffers reused by later plans, not checked-in binary art).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| LLM (judge model) response → `judgeEvaluator.ts` | The model's tool-call `arguments` are untrusted external input, even though it is "our own" judge model — an LLM can emit out-of-range numbers or a malformed shape despite the tool schema |
| `judgeEvaluator.ts` → persisted `detail` JSON | What gets written to disk/DB must never include secrets |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-3-03 | Tampering | `src/eval/judgeEvaluator.ts` `VerdictSchema.parse()` | high | mitigate | `zod` `.min(0).max(1)` numeric bounds on all three rubric dimensions before any judge score is trusted or persisted — an out-of-range or malformed tool-call argument is treated identically to a missing tool call (retry, then drop) |
| T-3-04 | Information Disclosure | `src/eval/judgeEvaluator.ts` `detail` JSON | medium | mitigate | Only `usage` (token counts, `cost.total`) is persisted from the model response; the raw request/response payload and the API key are never included in `detail` or logged |
| T-3-05 | Denial of Service (resource exhaustion) | `src/eval/judgeEvaluator.ts` retry loop | low | mitigate | `MAX_RETRIES = 2` bounded exponential backoff, then drop — no unbounded retry |
| T-3-09 | Tampering (config-time) | `src/eval/pixelmatchEvaluator.ts` dimension handling | medium | mitigate | Both images are unconditionally normalized to the stack viewport via `sharp` before decoding — a mismatched or adversarially-sized PNG cannot crash `pixelmatch` (Pitfall 1) |
