<objective>
Build the two pure infrastructure pieces the rest of Phase 3's scoring pipeline sits on top of: (1) a render capability that keeps a Playwright page open across multiple evaluators instead of closing it the instant a screenshot is taken (D3-17), and (2) the weighted-mean composite scorer with drop-and-renormalize semantics (D3-01/D3-03/D3-04, SCORE-01).

Purpose: `renderWithPage()` is the one new render seam the axe (EVAL-03) and DOM-presence (EVAL-02) evaluators need — Phase 2's `RenderPort.screenshot()` closes its browser/context in a `finally` block before returning, which structurally cannot expose a page. `composeScore()` is the one place SCORE-01's "composite from sub-scores" and D3-03's "renormalize, never zero a dropped evaluator" logic lives, decoupled from persistence (03-01) and from which evaluators actually ran (03-06).
Output: `src/render/renderWithPage.ts` (sibling to, not a replacement of, `playwrightRenderer.ts`); `src/pipeline/composite.ts`.
</objective>

    - A scenario-supplied custom weight map (not the default) is honored — passing `{pixelmatch: 0.7, dom: 0.1, axe: 0.1, judge: 0.1}` as `defaultWeights` changes the resulting `compositeScore` relative to the equal-weights case with the same raw scores.
  </behavior>
  <action>
    Create `src/pipeline/composite.ts` exporting: `export const DEFAULT_EVALUATOR_WEIGHTS: Record<string, number> = { pixelmatch: 0.25, dom: 0.25, axe: 0.25, judge: 0.25 };` (D3-02's default). `export interface EvalResult { evaluatorName: string; rawScore: number; dropped: false }`, `export interface DroppedResult { evaluatorName: string; dropped: true; reason: string }`, `export type EvaluatorOutcome = EvalResult | DroppedResult;`. `export interface CompositeResult { compositeScore: number | null; weightsUsed: Record<string, number>; }`. `export function composeScore(outcomes: EvaluatorOutcome[], defaultWeights: Record<string, number> = DEFAULT_EVALUATOR_WEIGHTS): CompositeResult` — filter `outcomes` to non-dropped survivors; if `survivors.length === 0` return `{ compositeScore: null, weightsUsed: {} }` immediately (Pitfall 5's explicit short-circuit, checked BEFORE any division); otherwise sum `defaultWeights[survivor.evaluatorName] ?? 0` across survivors into `survivorWeightSum`, then for each survivor compute `weight = (defaultWeights[survivor.evaluatorName] ?? 0) / survivorWeightSum`, accumulate `compositeScore += survivor.rawScore * weight`, and record `weightsUsed[survivor.evaluatorName] = weight`. This function has zero I/O and zero knowledge of which concrete evaluators exist — it only ever sees the `EvaluatorOutcome[]` array its caller (`evaluateRun`, 03-05-PLAN.md) constructs.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| `renderWithPage(input.url)` navigates to a caller-supplied URL and loads/executes whatever JS that page contains | In Phase 3 this is always a local fixture (`file://` or `node:http` test server); in later phases (deferred, out of scope) the same URL could point at a real agent-generated app with unbounded/hostile client-side JS |
| `composeScore`'s numeric inputs | Already-validated `[0,1]` raw scores from upstream evaluators — no untrusted external input crosses this boundary directly |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-3-02 | Denial of Service | `src/render/renderWithPage.ts` navigation | high | mitigate | Reuses `playwrightRenderer.ts`'s bounded `Promise.race` navigation-timeout pattern instead of an unbounded `waitUntil:"networkidle"`, so a page that never settles network activity can't hang the evaluation pipeline (03-RESEARCH.md Security Domain: DoS row) |
| T-3-08 | Denial of Service (accept, low likelihood) | `composeScore` | low | accept | Pure in-process arithmetic over an already-bounded, caller-constructed array; no external input, no loop over unbounded/attacker-controlled data — no mitigation needed beyond the existing type signature |

</threat_model>
