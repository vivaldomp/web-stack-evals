<objective>
Execute the D4-26 locked contract change: move `seq` ownership out of the caller and into `StoragePort.appendEvent`, so the agent adapter (Plans 04-07/08) and `runStack` can both append to a single run's log without coordinating a shared counter. Migrate the full blast radius the research enumerated — `ports.ts`, the SQLite adapter, `runStack.ts`'s local `let seq = 0`, and the test doubles — and add an interleaved-writer proof.

Purpose: Two producers now write one run's log. A caller-owned counter would collide the moment both append. Storage-owned atomic seq is the only correct contract.
Output: seq-storage-owned append boundary + a monotonicity test, with the whole suite green.
</objective>
