<objective>
Complete the Pi adapter's run loop by layering budget enforcement, honest usage-on-abort, guaranteed teardown, and the isolation decision onto the seams Plan 04-05 deliberately left in `src/agent/piAgentAdapter.ts`. 04-05 built the streaming adapter (single mockup prompt -> live `AgentEventDraft` stream) and retained the `session` handle plus three marked `// 04-06 SEAM` points; this plan fills them in.

A run that hits any one of the three ceilings must abort, keep the files the agent already wrote, and emit `benchmark_finished { status: "timeout" }` with cumulative usage that matches what the provider actually charged ‚Äî a capped run is a scored data point, not a discarded failure (AGENT-05, D4-01/02/15). No orphaned session or held port survives any exit path (D4-24).

It does NOT redesign 04-05's generator, re-run the mapper, or add the tool-op containment wrapper (that is the documented upgrade, not v1).
</objective>
