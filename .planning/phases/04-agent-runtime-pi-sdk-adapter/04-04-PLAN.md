<objective>
Build the pure Pi → canonical event mapping module (`src/agent/mapEvent.ts`): a per-run `createEventMapper(ctx)` factory whose returned mapper translates one Pi coding-agent SDK event into zero-or-more `AgentEventDraft`s, per the RESEARCH § "Event mapping table". This is the translation core the adapter (Plan 04-05) calls once per event; it holds no Pi session and does no I/O, so its entire behavior is validated from hand-authored fake Pi event objects.

Purpose: AGENT-04 ("Pi events normalized into the canonical `AgentEvent` stream") is fundamentally a faithful-translation problem. Isolating it as a pure function makes every non-deterministic/paid path (first-token timing, aborted-turn usage, retry attribution) deterministically testable with zero network and zero cost.
Output: `src/agent/mapEvent.ts` + `tests/mapEvent.test.ts`, both typecheck-clean and green.
</objective>

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-04-12 | Info-disclosure | narration persisted verbatim in `UnknownEvent.raw` | low | accept | The model can emit arbitrary text, but the provider API key never enters the prompt/context (D4-19 — it lives only in the orchestrator's in-memory `AuthStorage`), so no secret can surface in narration. | T-04-12b | Denial-of-service | unbounded `raw` payload size | low | accept | A pathological run could emit very large narration; acceptable for the single v1 row (D4-12: "log size is fine"). Revisit with truncation only if multi-row volume bites. |
