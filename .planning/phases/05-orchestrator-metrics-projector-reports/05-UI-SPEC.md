# Phase 5 — UI Design Contract

> Visual and interaction contract for the two user-facing surfaces of this phase:
> the **self-contained static HTML post-mortem report** (`results/<run_id>/report.html`)
> and the **terminal summary** printed by `run` / `report`. There is no interactive
> web app — the report is a single portable `.html` file built by a Node string
> template with base64-inline screenshots (D5-09), no runtime framework, no external
> asset fetch. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — static HTML string template (D5-04/D5-09); project is a Node/TS CLI, not React/Next/Vite, so the shadcn gate is **not applicable** |
| Preset | not applicable |
| Component library | none — hand-authored HTML + inline `<style>` block (must be self-contained; no CSS framework, no CDN) |
| Icon library | none — inline unicode glyphs / inline SVG only (no external icon fetch; portability is the whole point of D5-09) |
| Font | **system font stack only** (no web font — an embedded font would bloat the base64 payload and a linked font would break portability). Prose: `system-ui, -apple-system, "Segoe UI", Roboto, sans-serif`. Metric values, tool-call timeline, run_id: `ui-monospace, "SF Mono", "Cascadia Code", Menlo, monospace` |

**Self-containment invariant (non-negotiable, from D5-09):** the emitted `report.html`
must render identically with no network access and no sibling files — every image is a
`data:image/png;base64,…` URI, all CSS is one inline `<style>`, all JS (only the
collapse toggle) is one inline `<script>`. No `<link>`, no `<img src="./…">`, no `fetch`.

## Spacing Scale

Declared values (multiples of 4). This is a dense data report — favor the tighter rungs.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline gaps (glyph↔label), score-bar track padding |
| sm | 8px | Metric-row vertical rhythm, table cell padding |
| md | 16px | Card interior padding, gap between metric groups |
| lg | 24px | Card outer margin, gap between report sections |
| xl | 32px | Screenshot-triptych gap, header block padding |
| 2xl | 48px | Major section breaks (scorecard → metrics → timeline) |
| 3xl | 64px | Page top/bottom margin (max content width ~960px, centered) |

Exceptions: none.

## Typography

3 prose sizes + 1 display. Two weights only (400 regular, 600 semibold).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 12px | 600 | 1.4 |
| Heading | 20px | 600 | 1.25 |
| Display | 32px | 600 | 1.2 |

- **Body (14/400/1.5):** metric values, table cells, narration text, caveat prose.
- **Label (12/600/1.4):** section eyebrows, table column headers, metric names, status pills — uppercase with `letter-spacing: 0.05em` for eyebrows only.
- **Heading (20/600/1.25):** section titles (Scorecard, Screenshots, Metrics, Agent Timeline).
- **Display (32/600/1.2):** the composite score number in the scorecard, and the run-status pill headline.
- **Monospace exception:** metric numeric values, `backoff_wait_ms`, run_id, and every tool-call timeline entry render in the monospace family at Body size (14px) so digits align in columns. Deliberate, noted here.

## Color

Single light theme (a report is read, printed, and pasted into emails — a light surface is the safe, portable default). All values are literal hex baked into the inline `<style>`.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#f8fafc` | Page background field |
| Secondary (30%) | `#ffffff` | Cards / panels / table surfaces, `#e2e8f0` 1px borders |
| Accent (10%) | `#2563eb` | Score-bar fills, composite value, section-title underline, in-report anchors |
| Destructive | `#dc2626` | `build_failed` / `start_failed` status, error-stage marker, tool-call error count |

Accent reserved for: composite-score number, the four sub-score bar fills, the section-title underline rule, and internal jump links. **Never** the default text color and never generic borders.

Text & muted (not part of the 60/30/10 split — neutral ink):
- Primary text: `#0f172a` (slate-900). Muted labels / secondary metrics: `#64748b` (slate-500).

**Semantic status palette** (small, reserved strictly for the run-status pill and stage markers — a benchmark reports outcomes, so status color is load-bearing, not decoration):

| Status | Value | Meaning |
|--------|-------|---------|
| Scored / healthy | `#16a34a` green-600 | run completed with a generated screenshot + composite |
| Capped / timeout | `#d97706` amber-600 | run hit `timeout` — a valid scored data point (D5-05) |
| Failed stage | `#dc2626` red-600 (Destructive above) | `build_failed` / `start_failed` — still a scored data point (D5-05/D5-08) |

Status color appears **only** in the status pill, the failed-stage marker in the metrics table, and (as a thin left border) the caveat banner. It never colors body prose.

## Copywriting Contract

The report has no forms and no destructive actions (re-runs *append*, never overwrite — D5-10). The one interactive control is the collapsible agent-narration/tool-call timeline (D5-04). "Empty" and "error" states here mean **partial/failed-run rendering** (D5-05) and the **mockup-grounding caveat** (D5-01) — the states that make this report honest.

| Element | Copy |
|---------|------|
| Primary CTA (timeline disclosure) | Collapsed: **"Show agent timeline"** · Expanded: **"Hide agent timeline"** (native `<details>`/`<summary>`, open=false by default) |
| Empty state heading (no generated screenshot) | **"No screenshot captured"** |
| Empty state body (no generated screenshot) | "The run stopped at the **{failed_stage}** stage before the app rendered. The metrics below reflect everything the run did produce." (renders in the generated-screenshot slot of the triptych; expected + diff slots follow the same pattern when absent) |
| Empty state (composite is null) | Scorecard shows **"—"** in place of the composite number with a Label-sized note: "No composite — run did not reach evaluation." Sub-score bars that did compute still render; absent ones read "—". |
| Error state (`report` with unknown run_id) | Terminal, not HTML: **"Run '{run_id}' not found. Run `bench run --stack … --model … --scenario …` first, or `bench report --latest`."** (exit non-zero — harness error per D5-08) |
| Error state (`report --latest`, empty DB) | Terminal: **"No runs found. Run `bench run --stack … --model … --scenario …` to produce one."** |
| Destructive confirmation | none — no destructive action exists in this phase (append-only, D5-10) |

Report-specific state copy (beyond the template rows — required by locked decisions):

| State | Copy |
|-------|------|
| Run-status pill — scored | **"SCORED"** (green) |
| Run-status pill — capped | **"CAPPED · timeout"** (amber) |
| Run-status pill — failed | **"FAILED · {failed_stage}"** (red) — e.g. "FAILED · build" |
| Mockup-grounding caveat (D5-01, when image was not injected) | Banner near the scorecard, amber left-border: **"Visual-fidelity caveat: the agent had no mockup grounding for this run — the resolved model does not accept image input, so the expected screenshot was not shown to it. Scoring is unaffected (the judge diffs screenshots on its own vision model)."** |
| Backoff attribution note (D5-12, when `backoff_wait_ms` > 0) | Inline under the wall-time metric: **"Includes {backoff_wait_ms} ms waiting on provider rate-limit/backoff — surfaced separately, not counted as agent productive time."** |

**Terminal summary format (D5-03 — REPORT-01, printed after every `run` and by `report`):**

Compact block, monospace-aligned, no full metric dump. Exact contents and order:
1. **Header line:** `stack × model × scenario` and the run_id.
2. **Status line:** the status pill text (`SCORED` / `CAPPED · timeout` / `FAILED · {stage}`).
3. **Composite line:** composite score (rounded to presentation precision, e.g. 2 dp; underlying value stored verbatim per D-26), or `—` if null.
4. **Sub-scores line:** the four evaluator sub-scores (pixelmatch / dom / a11y / judge) as `name=value` pairs.
5. **Headline metric line (one line):** wall time, cost (USD), total tokens, iterations — e.g. `wall 42.1s · $0.037 · 18.4k tok · 6 iters`.
6. **Report path line:** `Report: results/<run_id>/report.html` (D5-07 — `run` always echoes the emitted path).

Rounding is presentation-only; stored values stay verbatim/unrounded (D-26).

## Report Layout (section order, top → bottom)

Single centered column, max-width ~960px. This ordering is the executor's contract.

1. **Header** — `stack × model × scenario`, run_id (monospace), timestamp, rep index, and the run-status pill (Display-adjacent).
2. **Mockup-grounding caveat banner** — only when the image was not injected (D5-01).
3. **Scorecard** — Display composite number + four sub-score bars (Accent fills over a `#e2e8f0` track), each bar labeled + value.
4. **Screenshots triptych** — expected / generated / diff, side by side (`xl` gap), each base64-inline (D5-09), each captioned. Absent slots use the empty-state copy above; a `build_failed` run is a scored data point, **not** an error screen (D5-05).
5. **Folded metrics** — complete table grouped: **Performance** (wall / build / startup / render, plus `backoff_wait_ms` called out separately per D5-12), **Engineering** (files created/edited, lines +/−), **Iteration** (iteration count, correction density per D5-11), **Tool calls by type** (read/write/edit/bash/grep/find/mcp with call + error counts). All folded from the log by the projector, never inline (TEL-02/D-24).
6. **Agent timeline** — collapsible `<details>` (default closed) holding verbatim narration + ordered tool-call timeline (D5-04, fed by D4-12 narration payloads), monospace.
