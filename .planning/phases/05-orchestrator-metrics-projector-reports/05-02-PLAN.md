<objective>
Deliver D5-01's token-saving intent for real (not cosmetic) via D5-14: a
capability probe that answers "does the resolved model accept image input?" and
an adapter gate that skips the mockup when it does not.

Two behaviors:
1. New `src/agent/modelCapabilities.ts` exporting `modelAcceptsImage(spec: AgentModelSpec): boolean`
   — reads the Pi model registry `input` field (`model.input?.includes("image")`).
   This is a SECOND `@earendil-works/pi-coding-agent` importer, so
   `tests/importBoundary.test.ts`'s allowlist is expanded IN THIS PLAN (else CI red).
2. Optional `injectImage?: boolean` (default = inject) on `AgentInput`, honored by
   `piAgentAdapter` with a one-line ternary on the `images` array. `assertAgentInput`'s
   `mockupBytes` requirement is UNCHANGED — the flag only controls whether it is sent.

The orchestrator (a later plan) sets `injectImage = modelAcceptsImage(model)` and
records the report caveat when false. This plan only builds the probe + gate + tests.

Purpose: without this, D5-01 is presentation-only — the adapter injects the mockup
unconditionally (piAgentAdapter.ts:306-315) and pays for image tokens a text-only
model discards. This closes RESEARCH gap A5.

Output: `src/agent/modelCapabilities.ts`, `AgentInput.injectImage`, the adapter
ternary, and the golden unit tests + updated import-boundary allowlist.
</objective>
