<objective>
Build the self-contained static HTML post-mortem report: a single pure function
`renderReport(db, runId, resultsRoot = "results"): string` in `src/reports/renderReport.ts`
that turns one stored benchmark run into a portable `.html` string, using template
literals only (no runtime framework, no CDN). It reads ONLY the results DB rows and
on-disk artifact PNGs, so `report <id>` regenerates it offline (REPORT-02, CLI-02).

Screenshots (expected / generated / diff) are base64-inlined as `data:` URIs (D5-09);
the scorecard shows the composite plus four sub-score bars; a complete folded-metrics
table (Performance / Engineering / Iteration / Tool-calls) is rendered from the
projection tables; a mockup-grounding caveat banner appears when the agent was not
shown the mockup (D5-01); and a collapsible agent-narration + tool-call timeline is
fed by the verbatim narration D4-12 persisted in `unknown` events. Partial and failed
runs render as scored data points with empty-state copy, never an error screen (D5-05).

Purpose: the shareable, self-contained artifact a human reads/prints/emails to judge a
run — the honest post-mortem the whole benchmark exists to produce.
Output: `src/reports/renderReport.ts` exporting `renderReport`, proven by `tests/renderReport.test.ts`.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| agent output → report HTML | Agent narration (unknown events), tool `argsSummary`, and `run_id` are untrusted text interpolated into the HTML string — the report's XSS surface. |
| stored DB + artifacts → report | The report reads only DB rows + on-disk PNG bytes it base64-inlines; there is no live network fetch, so a run cannot beacon or exfiltrate on open. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-01 | Tampering | `renderReport` interpolation of agent narration / run_id / tool argsSummary / failed_stage into HTML | high | mitigate | Single `esc()` HTML-escapes (&, <, >, ", ') every interpolated untrusted value before it enters the template; a test feeds a script element + onerror image in narration/run_id/tool args and asserts the output carries escaped entities, not live markup (Task 1 escaping case, blocking gate at ASVS L1). |
| T-05-02 | Information Disclosure | any external `<link>` / external image or script source / `fetch` in the emitted HTML | medium | mitigate | Self-containment invariant (D5-09): images are `data:` URIs, styling is one inline block, collapse uses native details/summary (no external script). A test asserts zero external references and every image is `data:`. |
| T-05-SC | Tampering | npm/pip/cargo installs | low | accept | This plan adds no packages — base64 embedding uses `node:fs` `readFileSync().toString("base64")`; no new dependency (05-RESEARCH §Package Legitimacy Audit is empty by construction). |
</threat_model>
