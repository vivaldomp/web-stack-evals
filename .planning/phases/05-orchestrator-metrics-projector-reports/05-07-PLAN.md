<objective>
The user-facing `bench` CLI: two subcommands (`run`, `report`) over the already-built orchestrator (05-06) and HTML report renderer (05-05). `run` executes one benchmark row from named spec flags, prints the compact terminal summary (REPORT-01 / D5-03), auto-writes `results/<run_id>/report.html` (D5-07), and maps exit codes per D5-08. `report [<run_id>] [--latest]` regenerates the summary + report for a stored run (CLI-02 / D5-06).

Purpose: give a human one command to produce and one command to regenerate the full result set, with honest partial-run rendering (D5-05) and a traversal-safe arg surface.

Output: `src/cli/cli.ts` (testable core), `src/cli/index.ts` (bin shim), `tests/cli.test.ts`, and `package.json` bin/script wiring. Thin wrapper — no business logic beyond arg parsing, wiring, and summary formatting (D-23; all folds/scoring live upstream).
</objective>
